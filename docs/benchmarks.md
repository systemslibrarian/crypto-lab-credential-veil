# Benchmarks

Reproducible numbers behind the README's performance claims. Regenerate with:

```bash
npm run bench
```

Not a CI gate — wall-clock numbers flake across runners; the contract is that the
table below can be regenerated by anyone with one command, on hardware they name.

## Recorded run

- **Date:** 2026-07-17
- **Hardware:** Intel Core Ultra 7 256V (laptop class), Windows 11
- **Runtime:** Node 26 / V8 (same engine class as Chromium in the browser; expect
  Firefox (SpiderMonkey) and Safari (JavaScriptCore) to differ — the in-browser a11y
  suite exercises all three and the demo stays interactive on each because proofs run
  in a Web Worker)
- **Workload:** the demo's own six-field credential, `@noble/curves` 1.9.7, pure JS,
  no WASM, nothing precomputed

| Operation | mean | min–max (10 iter) |
|---|---|---|
| BBS sign (6 messages) | 83 ms | 56–136 ms |
| Present — reveal 1 of 6 | 125 ms | 103–183 ms |
| Present — reveal all 6 | 75 ms | 64–80 ms |
| Verify presentation (1 of 6) | 74 ms | 70–78 ms |
| Verify presentation (all 6) | 66 ms | 62–69 ms |
| Prove age ≥ 18 (15-bit linked range proof) | 352 ms | 316–401 ms (3 iter) |
| Verify age proof | 1,102 ms | 1,076–1,151 ms (3 iter) |

## Reading the numbers

- **Presenting with more hidden fields costs more** — each undisclosed message adds a
  blinded term to the proof. Reveal-1-of-6 carries five hidden messages and is the
  slowest presentation; reveal-all carries none.
- **The age proof's cost is in verification**, not proving: the verifier recomputes 15
  Chaum–Pedersen OR-proof commitment pairs (≈60 G1 scalar multiplications) plus the
  pairing check. The prover reuses its own nonces and does roughly half the group work.
- **Cold start vs warm:** the first operation in a fresh tab/worker also pays
  `create_generators` and curve-table initialization (a few hundred ms, once). The
  table above is warm-run.
- **Proof sizes** (fixed by construction, not benchmarked): BBS presentation proof =
  `3·48 + (4+U)·32` bytes, where U is the hidden-message count — 272 bytes revealing
  all six fields up to 464 bytes revealing none; the age proof adds one 48-byte
  commitment, two 32-byte scalars, and 15 bit proofs of 144 bytes each on top of a
  reveal-nothing BBS proof (≈2.7 kB total).
- **Bundle size** is printed by `npm run build` (currently ~81 kB main + ~89 kB worker,
  ~31 kB gzipped main); track it there rather than here.
